# Audit Module

Audit trail tracking. Pass a `RepositoryLike` (mongokit / prismakit /
your own kit) for persistence, or omit for in-memory dev.

## Setup

```typescript
import { auditPlugin } from '@classytic/arc/audit';
import { Repository } from '@classytic/mongokit';
import mongoose, { Schema } from 'mongoose';

// Development: in-memory (default, no repository passed)
await fastify.register(auditPlugin, { enabled: true });

// Production: any RepositoryLike
const AuditModel = mongoose.model(
  'ArcAuditEntry',
  new Schema({}, { strict: false, timestamps: false, _id: false }),
  'audit_logs',
);

await fastify.register(auditPlugin, {
  enabled: true,
  repository: new Repository(AuditModel),
});
```

TTL / retention are owned by your DB — create a TTL index on `timestamp`
(Mongo), a scheduled DELETE job (SQL), or equivalent. Arc no longer
creates indexes automatically.

## Configuration

```typescript
interface AuditPluginOptions {
  enabled?: boolean;              // Enable auditing (default: false)
  repository?: RepositoryLike;    // Persistent backend (mongokit / prismakit / custom)
  customStores?: AuditStore[];    // Extra store implementations (fan-out)
  autoAudit?: {
    operations?: Array<'create' | 'update' | 'delete' | 'restore'>;
    perResource?: boolean;        // opt-in per resource via defineResource({ audit })
  };
}
```

- **No repository passed** → in-memory store (dev/test)
- **`repository` passed** → all writes go to the repo's backing DB
- **`customStores`** → fan-out to additional sinks (e.g. Loki, S3)

## Using Audit Logger

```typescript
// Log CRUD actions
await fastify.audit.create('product', product._id, product, request.auditContext);
await fastify.audit.update('product', id, beforeDoc, afterDoc, request.auditContext);
await fastify.audit.delete('product', id, deletedDoc, request.auditContext);
await fastify.audit.restore('product', id, restoredDoc, request.auditContext);

// Log custom actions
await fastify.audit.custom('product', id, 'price_changed', {
  oldPrice: 100,
  newPrice: 150,
}, request.auditContext);
```

## Audit Context

Automatically populated from request:

```typescript
interface AuditContext {
  user?: UserBase;
  organizationId?: string;
  requestId?: string;
  ipAddress?: string;
  userAgent?: string;
  endpoint?: string;           // e.g., 'POST /api/products' (auto-set after response)
  duration?: number;           // Request duration in ms (auto-set after response)
}

// Access in handlers
const context = request.auditContext;
```

**Note:** `endpoint` and `duration` are populated in Fastify's `onResponse` hook, so they're available in audit entries written during response lifecycle.

## Audit Entry Structure

```typescript
interface AuditEntry {
  id: string;
  timestamp: Date;
  action: 'create' | 'update' | 'delete' | 'restore' | 'custom';
  resource: string;
  documentId: string;
  before?: Record<string, unknown>;
  after?: Record<string, unknown>;
  metadata?: Record<string, unknown>;
  context: AuditContext;
}
```

## Querying Audit Logs

```typescript
const entries = await fastify.audit.query({
  resource: 'product',
  documentId: 'product-123',
  action: 'update',
  from: new Date('2024-01-01'),
  to: new Date(),
  limit: 100,
});
```

## Custom Store Implementation

```typescript
import type { AuditStore, AuditEntry, AuditQueryOptions } from '@classytic/arc/audit';

class CustomAuditStore implements AuditStore {
  async log(entry: AuditEntry): Promise<void> {
    // Store the entry
  }

  async query(options: AuditQueryOptions): Promise<AuditEntry[]> {
    // Query entries
    return [];
  }

  async close(): Promise<void> {
    // Cleanup
  }
}

// Use it
await fastify.register(auditPlugin, {
  enabled: true,
  customStores: [new CustomAuditStore()],
});
```

## Deployment Guide

| Setup | Backend | Notes |
|-------|---------|-------|
| Dev / test | in-memory (omit `repository`) | Fast, no persistence |
| Multi-instance | `repository: new Repository(AuditModel)` | Shared, persistent |
| Compliance | same + TTL index on `timestamp` | Auto-cleanup after retention |
