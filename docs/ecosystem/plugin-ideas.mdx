# Plugin Ideas

Community plugin ideas for the Arc ecosystem. Each entry includes the plugin type, difficulty, and a sketch of how it would work with Arc's extension points.

**Want to build one?** See [Building Plugins](./building-plugins) for the full guide.

## Tier 1 — High Impact, Ready to Build

### `arc-plugin-file-upload`
**Type:** Arc Plugin (forRoot/forResource) | **Difficulty:** Medium

S3-compatible file upload with presigned URLs, image processing, and per-resource configuration.

```typescript
const uploads = createPlugin('file-upload', {
  forRoot: async (fastify, { s3Client, bucket }) => {
    fastify.decorate('storage', new StorageService(s3Client, bucket));
  },
  forResource: (config, { fields, maxSize, allowedTypes }) => ({
    routes: [
      { method: 'POST', path: '/:id/upload', handler: 'handleUpload' },
      { method: 'GET', path: '/:id/files', handler: 'listFiles' },
    ],
    middlewares: {
      create: [validateFileFields(fields, { maxSize, allowedTypes })],
    },
  }),
});
```

### `arc-plugin-search`
**Type:** Arc Plugin (forRoot/forResource) | **Difficulty:** Medium

Full-text search powered by Elasticsearch, Meilisearch, or Typesense. Auto-indexes resources on create/update/delete.

```typescript
const search = createPlugin('search', {
  forRoot: async (fastify, { engine, url, apiKey }) => {
    const client = createSearchClient(engine, { url, apiKey });
    fastify.decorate('search', client);
  },
  forResource: (config, { searchableFields, filters }) => ({
    hooks: [
      { operation: 'create', phase: 'after', handler: (ctx) => index(ctx) },
      { operation: 'update', phase: 'after', handler: (ctx) => index(ctx) },
      { operation: 'delete', phase: 'after', handler: (ctx) => deindex(ctx) },
    ],
    routes: [
      { method: 'GET', path: '/search', handler: 'search' },
    ],
  }),
});
```

## Tier 2 — Medium Impact, Great for Contributors

### `arc-plugin-notifications`
**Type:** Arc Plugin | **Difficulty:** Medium

Multi-channel notifications (email, SMS, push, in-app) triggered by resource events.

```typescript
const notifications = createPlugin('notifications', {
  forRoot: async (fastify, { channels }) => {
    // channels: { email: resendClient, sms: twilioClient, push: fcmClient }
  },
  forResource: (config, { templates }) => ({
    hooks: [{
      operation: 'create', phase: 'after',
      handler: (ctx) => notify(templates.onCreate, ctx.result),
    }],
  }),
});
```

### `arc-pipeline-rate-limit`
**Type:** Pipeline Step | **Difficulty:** Easy

Per-resource, per-operation rate limiting as a pipeline guard.

```typescript
import { rateLimit } from 'arc-pipeline-rate-limit';

defineResource({
  name: 'api-key',
  pipe: {
    create: pipe(rateLimit({ max: 5, windowMs: 60_000 })),
    list: pipe(rateLimit({ max: 100, windowMs: 60_000 })),
  },
});
```

1. Pick an idea (or propose your own)
2. Read [Building Plugins](./building-plugins)
3. Build and publish to npm
4. Submit a PR to add it to the [Arc Plugin Registry](https://github.com/classytic/arc-plugins)

All contributions welcome — from simple presets to full database adapters.
