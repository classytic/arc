# HTTP Test Harness

`createHttpTestHarness` auto-generates CRUD, permission, and validation tests against a live arc app. It exercises the full request lifecycle — routing, auth, permissions, pipeline, action endpoints, field masking, response envelope — so a single call gives every resource baseline coverage for free.

## Setup

```typescript
import { beforeAll, afterAll } from 'vitest';
import { createTestApp, createHttpTestHarness } from '@classytic/arc/testing';
import jobResource from '../src/resources/job/job.resource.js';

let ctx;
beforeAll(async () => {
  ctx = await createTestApp({ resources: [jobResource], authMode: 'jwt' });
  ctx.auth.register('admin',  { user: { id: '1', roles: ['admin']  } });
  ctx.auth.register('member', { user: { id: '2', roles: ['member'] } });
});
afterAll(() => ctx.close());

createHttpTestHarness(jobResource, () => ({
  app: ctx.app,
  auth: ctx.auth,
  adminRole: 'admin',
  fixtures: {
    valid:   { title: 'Test Job' },
    update:  { title: 'Updated Job' },
    invalid: {},                     // missing required fields
  },
})).runAll();
```

This generates ~16 tests:

- **CRUD** — `POST` create, `GET` list, `GET /:id`, `PATCH` or `PUT` (or both) `/:id`, `DELETE /:id`, 404 probes
- **Permissions** — unauthenticated 401 **only for protected ops**, admin succeeds on every op
- **Validation** — invalid payload is rejected with 4xx + `success: false`

The harness reads `resource.permissions` to decide which ops get a "without auth should 401" assertion. Ops declared with `allowPublic()` (or with no permission at all) are skipped — arc correctly serves them to anonymous requests, so asserting 401 would produce a false failure.

The harness also iterates every update verb your resource exposes. `updateMethod: 'both'` runs PATCH AND PUT tests; single-method resources run one.

## Deferred vs eager options

Options can be a plain object (app ready at module level) OR a getter (app from async `beforeAll`). Prefer the getter form — test setup almost always needs `beforeAll`:

```typescript
// Eager (rare)
createHttpTestHarness(resource, { app, auth, adminRole: 'admin', fixtures }).runAll();

// Deferred (recommended)
createHttpTestHarness(resource, () => ({
  app: ctx.app, auth: ctx.auth, adminRole: 'admin', fixtures
})).runAll();
```

## Selective runs

`runAll()` runs CRUD + Permissions + Validation. Run them in isolation:

```typescript
const harness = createHttpTestHarness(resource, () => options);
harness.runCrud();
harness.runPermissions();
harness.runValidation();
```

## What it respects

The harness reads the resource definition:

- `disabledRoutes: ['delete']` → delete tests are skipped
- `disableDefaultRoutes: true` → no CRUD tests (useful for action-only or custom-routes-only resources)
- `updateMethod: 'PUT' | 'PATCH' | 'both'` → exercises every verb the resource exposes (`'both'` runs PATCH AND PUT)
- `permissions[op]` → determines whether a "401 without auth" assertion is emitted (public ops skip it)

## Action endpoints

Since v2.11, arc's action routes (`POST /:id/action`) share the preHandler chain with CRUD. The HTTP harness doesn't auto-generate action tests (action payloads are resource-specific), but action endpoints hit the same auth/permission/pipeline middleware, so CRUD harness coverage already validates the chain.

For action-specific assertions, use `createTestApp` directly:

```typescript
import { createTestApp, expectArc } from '@classytic/arc/testing';

const ctx = await createTestApp({ resources: [orderResource], authMode: 'jwt' });
ctx.auth.register('admin', { user: { id: '1', roles: ['admin'] } });

const res = await ctx.app.inject({
  method: 'POST',
  url: `/orders/${orderId}/action`,
  headers: ctx.auth.as('admin').headers,
  payload: { action: 'approve' },
});
expectArc(res).ok().showsField('status');
```

## API

### `createHttpTestHarness(resource, options | optionsGetter)`

- `resource` — a `defineResource(...)` result
- `options` / `optionsGetter` — `HttpTestHarnessOptions` or a function returning one

Returns `HttpTestHarness` with `runAll()`, `runCrud()`, `runPermissions()`, `runValidation()`.

### `HttpTestHarnessOptions<T>`

```typescript
interface HttpTestHarnessOptions<T> {
  app: FastifyInstance;         // Fastify instance (must be ready)
  auth: TestAuthProvider;       // from createTestApp or createJwtAuthProvider
  adminRole: string;            // role name registered on `auth` with full access
  fixtures: {
    valid: Partial<T>;          // body for POST create
    update?: Partial<T>;        // body for PATCH/PUT update (defaults to `valid`)
    invalid?: Partial<T>;       // body that should be rejected
  };
  apiPrefix?: string;           // URL prefix (default: "")
}
```
