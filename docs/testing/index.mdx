# Testing

Arc ships a small, focused test toolkit at `@classytic/arc/testing`. Three primary entry points; pick by what you're testing.

```bash
npm install -D vitest mongodb-memory-server
```

## Which harness when

| You're testing… | Use | Why |
|---|---|---|
| **Framework behavior** — routes, auth, permissions, pipeline, action endpoints, field masking | [`createHttpTestHarness`](./http-harness) | Auto-generates CRUD + permission + validation tests against a live app. Exercises the same code path production hits. |
| **Custom scenarios** — one-off flows, cross-resource interactions, app-lifecycle edge cases | [`createTestApp`](./test-app) | Turnkey Fastify + in-memory Mongo + auth provider + fixture tracker, ~3 lines of setup. You drive `app.inject()` directly. |
| **Adapter contracts** — your custom `RepositoryLike` / `Storage` implementation | `runStorageContract` | DB-agnostic conformance suite. No Mongoose assumption. |

Everything else composes with one of these three — [auth sessions](./auth-session), [fixtures](./fixtures), and [assertions](./assertions) are shared primitives consumed by all three entry points.

## Quick Start — HTTP harness

```typescript
import { beforeAll, afterAll } from 'vitest';
import { createTestApp, createHttpTestHarness } from '@classytic/arc/testing';
import jobResource from '../src/resources/job/job.resource.js';

let ctx;
beforeAll(async () => {
  ctx = await createTestApp({ resources: [jobResource], authMode: 'jwt' });
  ctx.auth.register('admin', { user: { id: '1', roles: ['admin'] } });
});
afterAll(() => ctx.close());

createHttpTestHarness(jobResource, () => ({
  app: ctx.app,
  auth: ctx.auth,
  adminRole: 'admin',
  fixtures: {
    valid: { title: 'Test Job' },
    update: { title: 'Updated Job' },
    invalid: {},
  },
})).runAll();
```

Generates ~16 tests: CRUD happy-path, 404 probes, unauthenticated 401s, admin access, invalid-payload rejection.

## Quick Start — custom scenarios

```typescript
import { createTestApp, expectArc } from '@classytic/arc/testing';

const ctx = await createTestApp({ resources: [jobResource] });
ctx.auth.register('admin', { user: { id: '1', roles: ['admin'] } });

const res = await ctx.app.inject({
  method: 'GET',
  url: '/jobs',
  headers: ctx.auth.as('admin').headers,
});
expectArc(res).ok().paginated({ page: 1, limit: 20 });

await ctx.close();
```

## Quick Start — adapter contract

```typescript
import { runStorageContract } from '@classytic/arc/testing';
import { createMyStorage } from '../src/storage/myStorage.js';

runStorageContract({
  async setup() {
    const storage = createMyStorage();
    return { storage, teardown: async () => storage.close() };
  },
});
```

Runs the full Storage contract (upload, download, delete, metadata, multi-part, error cases) against your implementation.

## What's NOT here

- **Mongoose-only harnesses.** The pre-rewrite `TestHarness` advertised DB-agnosticism but was hard-bound to Mongoose. Removed in 2.11. Write targeted unit tests instead, or use `createTestApp({ db: false })` with your own adapter.
- **Separate JWT vs Better Auth providers.** Unified into [`TestAuthSession`](./auth-session) — one abstraction, register roles with `user` (JWT payload) or `token` (pre-signed) or both.
